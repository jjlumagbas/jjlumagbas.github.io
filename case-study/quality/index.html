Quality People's platform uses clever marketing and big-data analytics to help schools find teachers who best fit open positions.

After achieving traction with several schools, the next step was to enable schools to manage multiple hiring pipelines. This required changes to several platform front-ends, as well as the underlying data model and API structure.

Extra challenge: the original devs had moved on, so we had to do some code-diving to uncover any undocumented assumptions made in the current implementation. We didn't want to unintentionally break some other part of the platform!

I worked with VP of Product to nail down the changes required, and then drafted user stories, feature specs, and mockups that developers needed to implement the changes.
